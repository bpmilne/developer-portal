Description

Notifies the consumer application's subscribed Webhook URL of transaction status changes for all transactions initiated by the given consumer application.

HTTP Method: POST Content-Type: 'application/json'

Request

Param Description
Type The type of Notification
Subtype The subtype or secondary type of Notification
Id Transaction identifier
Created Date the transaction was created by Dwolla
Triggered Date of the event triggering the notification
Value Transaction status. Fees bound to any transaction have the same status as the parent transaction.  

Possible values: 'processed', 'pending', 'cancelled', 'failed', 'reclaimed'
Transaction JSON object containing details of the Transaction
Status Current Transaction status, same as 'Value' in the parent JSON object
ClearingDate The expected clearing date of the transaction
Notes The notes associated with the money transfer when sent. Returned only for transactions of Type 'money_received' or 'money_sent', absent otherwise. Limited to 250 characters.
Destination The recipient of the money
Id Account identifier of the receiving user
Name Name of the receiving user
Type Type of the receiving user
Source The sender of money
Id Account identifier of the user sending the money
Name Name of the user sending the money
Type Type of the user sending the money
Amount Amount of the transaction.
SentDate Date and time when the transaction was created.
Fees Array of fees associated with this transaction
Type Transaction Type
Id Current Transaction status, same as 'Id' in the parent JSON object
Metadata JSON object containing metadata information 
POST https://www.example.com/payments/webhook
Content-Type: application/json
