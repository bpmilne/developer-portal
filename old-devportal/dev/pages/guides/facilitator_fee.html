Facilitator Fees

Overview

The facilitator fee is the application feature which allows a percentage or flat rate amount removed from every transaction as a fee and sent to the creator of the application / the Dwolla ID associated with their account. This fee comes out of the transaction total and is not appended to the transaction total itself. By default, this application feature is available to test against in our sandbox environment.

Facilitator fees can be expressed in one of two ways:

  • Percentage: The fee is a percentage of the transaction that the facilitator receives not exceeding 50% of the transaction total.
  • Fixed Rate: The fee is a flat rate greater than $0.01 and not exceeding 50% of the transaction total.
Note: facilitator fee is a premium application feature. To try it out, please contact us or call 1-888-289-8744.

Setting Facilitator Fees

The facilitator fee can either be set in your Dwolla application settings, or per each transaction via the API. Please note that if custom fees are specified via the API they will override the application settings set via the Dwolla web portal.

Dwolla Application Settings

  1. To enable facilitator fee, contact us through this short form or by phone at 1-888-289-8744.

  2. Click here and log in to view your list of applications, then hit "Edit Features".

  3. The "Facilitator Fee" section will allow you to set your fee as either a fixed rate, or as in the example below, a percentage. Hit "Update Features" when you are finished making your changes.

Using the API / Overriding

If certain products or services require you to list different or multiple facilitator fees, the Dwolla API can be used in order to override the application settings and provide different fees for the transaction being sent. The fees can be modified/set via the facilitatorAmount and additionalFees parameters.

Parameter Details
Parameter Description
facilitatorAmount Amount of the facilitator fee to override. Only applicable if the facilitator fee feature is enabled. If set to 0, facilitator fee is disabled for transaction. Cannot exceed 50% of the 'amount'.
additionalFees Array of additional facilitator fees
--> destinationId Facilitator's Dwolla ID
--> amount Amount of fee

The following endpoints currently support the facilitatorAmount and additionalFees parameters:

  • transactions/send
  • requests/request
  • payment/pay (Off-site Gateway)
  • offsitegateway/checkouts (Off-site Gateway)
Example

Let's say that your application is currently configured to take a 15% fee in order to compensate for the delivery of your services. You would like to offer your product via digital download and waive your 15% fee for the transaction.

A standard HTTP request, with the 15% fee configured in your application settings would look like this:

HTTP Request
POST https://www.dwolla.com/oauth/rest/requests/
Content-Type: application/json

{
    "oauth_token": "token_goes_here",
    "sourceId": "812-133-7777",
    "amount": "24.99"
}

In order to waive the fee, add the facilitatorAmount parameter and set it to 0 as such:

HTTP Request
POST https://www.dwolla.com/oauth/rest/requests/
Content-Type: application/json

{
    "oauth_token": "token_goes_here",
    "sourceId": "812-133-7777",
    "amount": "24.99",
    "facilitatorAmount": "0"
}

Using the Dwolla Libraries

If you are using our libraries, the examples below will take you step by step and show you how to pass facilitator fees to the transactions/send and transaction/request endpoints:

<?php
// Include the Dwolla REST Client
require '../lib/dwolla.php';

// Include any required keys
require '_keys.php';

// Instantiate a new Dwolla REST Client
$Dwolla = new DwollaRestClient();

// Seed a previously generated access token
$Dwolla->setToken($token);

// Send $1.00 to a Dwolla ID with a $5.00 facilitator fee. 
$transactionId = $Dwolla->send($pin, '812-734-7288', 1.00, 'Dwolla', 'Thanks for the coffee!', 5.00);

// Request $10.00 from a Dwolla ID with a $1.00 facilitator fee. 
$requestId = $Dwolla->request('812-234-4567', 10.00, 'Dwolla', 'Payment for lunch!', 1.00);
?>

# Include the Dwolla gem
require 'rubygems'
require 'pp'
require '..\lib\dwolla'

# Include any required keys
require './_keys.rb'

# Instantiate a new Dwolla User client
# And, seed a previously generated access token
Dwolla::token = @token
Dwolla::debug = true
Dwolla::sandbox = true

# Send money ($1.00) to a Dwolla ID with a $1.50 facilitator fee. 
transactionId = Dwolla::Transactions.send({:destinationId => '812-626-8794', :amount => 1.00, :pin => @pin, :facilitatorAmount => '1.50'})

# dwolla-ruby does not support the `transactions/request` endpoint at this time.

from dwolla import transactions, constants, request

# Configure the library (change these)
constants.sandbox=False

constants.client_id = "..."
constants.client_secret = "..."
constants.access_token = "..."
constants.pin = 1234

# Example 1: Send $5.50 to a Dwolla ID, with a $2.00 facilitator fee.

print(transactions.send('812-197-4121', 5.50, additionalFees=[{"destinationId": "812-122-1234", "amount": 2.00}]))

# Example 2: Request $5.50 from 812-740-3809 with a 2.00 facilitator fee. 

print(request.create('812-740-3809', 5.50, additionalFees=[{"destinationId": "812-122-1234", "amount": 2.00}]))

var dwolla = require('../lib/dwolla');
var c = require('../config');

// Pass the facilitatorAmount parameter in with 2.00
var params = { facilitatorAmount: 2.00 };

// Send $1.00 to a Dwolla ID with a $2.00 facilitator fee. 
dwolla.send(c.token, c.pin, '812-626-8794', 1.00, params, function(err, tran) {
  console.log(err);
  console.log(tran);
});

// Request $10.00 from a Dwolla ID with a $2.00 facilitator fee. (params is reused)
dwolla.request(c.token, c.pin, '812-626-8794', 10.00, params, function(err, tran) {
  console.log(err);
  console.log(tran);
});
