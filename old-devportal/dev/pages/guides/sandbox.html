Sandbox Environment

Overview

While you're developing your application, it's important to be able to test it against Dwolla frequently. The Sandbox environment is a complete replica of the main production environment, supporting all of the same API endpoints. Applications should be built and tested against the sandbox before being used in production.

Differences from production

  • The Sandbox only contains fake data (your production account won't be available here)
  • No actual money is touched or moved!
  • The Dwolla web interface is available at https://uat.dwolla.com/
  • All API endpoints have a base URL of https://uat.dwolla.com/ instead of https://www.dwolla.com

Getting started!

In order to start moving money and begin playing around in the Sandbox, you'll need to do the following:

  1. Create a Master sandbox account. »
  2. Create API credentials » using the master sandbox account.
  3. Use your master account and the Sandbox Admin interface to create accounts » with funds and attached bank accounts.

Creating a master sandbox account

  1. In order to get started with sandbox, a developer must first make an account by going to the aforementioned URL or by clicking here. You will want to use a real e-mail address so that you may confirm your account and/or receive notifications with regards to it.

  2. Choose an account type. This tutorial covers registering through the personal account route, however, all account types work on sandbox.

  3. You can enter false information for everything in this form (including SSN). We advise you to remember your pin, as this will be necessary for making calls to the API.

  4. After submitting this form, you should receive confirmation e-mails prepended with "TEST" in the subject confirming your registration. You will be taken to your account in the "TEST" environment. You can verify that you are in sandbox by looking for the orange advisory bar at the top of the page. It is there at all times when you browse the Dwolla web portal. Important note: Your master account will not have an attached funding source, or a way to add a funding source. Please refer to the "Sandbox Admin" section below on adding additional test accounts.

Sandbox Admin

In order to test how your application modifies account details or moves money amongst them, you can make multiple sandbox users with balances and attached bank accounts. Let's make a few right now with the sandbox admin tool:

  1. Log in with your master account, then navigate to https://sandbox-uat.dwolla.com or click here. If this is your first time logging into sandbox, you will be prompted with an OAuth permission request dialog. You must accept in order to use the sandbox admin tool.

  2. You will then be taken to the sandbox admin tool interface, where you can create additional accounts. This must be done by uploading a .csv file with the account details. More information is available in the sandbox admin tool, as well as a downloadable sample file. Make a .csv with a few accounts and upload it to the site.

  3. After uploading your *.csv, the initial balances you specified should appear under the account. From here, you can activate/deactivate accounts, as well as process, fail, cancel, or reclaim any transactions amongst them made through either the Dwolla web portal or the API. Remember, in order to access these accounts via the API you must use the https://uat.dwolla.com/ prefixed endpoints.

Remember to save the Dwolla account information used in the csv upload process. Once these accounts are created you can use the email address and password to login to the dwolla.com dashboard.

Dwolla Libraries; Enabling Sandbox

The Dwolla libraries support the UAT sandbox, making it easier for developers to test their applications without them having to tediously modify multiple procedures and variables. The code examples below give specifics on how to enable sandbox using our most popular libraries:

Within Library

<?php
/**
 * An official Guzzle based wrapper for the Dwolla API.
 * Fully modular; instantiate a subclass which contains only the group of endpoints
 * which you want to use, and not those that you don't.
 * The following intantiates the OAuth class, 
 * and enables sandbox mode via the settings class
 */
// We need the OAuth class in order to do anything
require '../vendor/autoload.php';
$OAuth = new Dwolla\OAuth();
// The Settings class is accessible from all sub-classes
$OAuth->settings->sandbox = true;
?>

# Import everything from the dwolla package
from dwolla import *

# Configure the library (change these)
constants.sandbox=True

constants.client_id = "..."
constants.client_secret = "..."
constants.access_token = "..."


# Example 1: Get basic information for a user via
# their Dwolla ID.

print(accounts.basic('812-202-3784'))

# Include the Dwolla gem
require 'rubygems'
require 'pp'
require 'dwolla'

# Include any required keys
require './_keys.rb'

# Instantiate a new Dwolla User client
# And, seed a previously generated access token
Dwolla::token = @token
Dwolla::api_key = @api_key
Dwolla::api_secret = @api_secret

# Enable Sandbox
Dwolla::sandbox = true

// Include the library and config information
var dwolla = require('../lib/dwolla');
var c = require('../config');

// Enable sandbox
dwolla.sandbox = true;

Special Instructions

Some libraries require a small amount of setup in order to use with Sandbox. Below are instructions for configuring each library for sandbox support.

dwolla-ios and dwolla-osx

Navigate to ./DwollaOAuthLib/Constants.m to view an application example, and make the following edits on line 9:

// No sandbox support
static NSString *DWOLLA_API_BASEURL = @"https://www.dwolla.com/oauth/rest";

// Sandbox support
static NSString *DWOLLA_API_BASEURL = @"https://uat.dwolla.com/oauth/rest";

dwolla-scala

In order to use sandbox with our Scala library, you must set the DWOLLA_API_HOST environment variable to uat.dwolla.com. Here's how you would do this in the most commonly used shells:

REM Make sure there are no spaces in between the '=' character
set DWOLLA_API_HOST=uat.dwolla.com

# 'uat.dwolla.com' must be enclosed in ' characters
$env:DWOLLA_API_HOST='uat.dwolla.com'

# Session only
set DWOLLA_API_HOST=uat.dwolla.com

# Permanent (stored in config/not recommended)
export DWOLLA_API_HOST=uat.dwolla.com

# Make sure there are no spaces in between the '=' character
DWOLLA_API_HOST=uat.dwolla.com

dwolla-java

Navigate to ./samples/src/main/java/com/example/App.java to view an application example, and make the following edits on lines 16-18:

public class App {
    private final static String baseUrl = "https://www.dwolla.com/oauth/rest";
    // Sandbox mode: private final static String baseUrl = "https://uat.dwolla.com/oauth/rest";
    private final static String applicationKey = "";

public class App {
    // private final static String baseUrl = "https://www.dwolla.com/oauth/rest";
    private final static String baseUrl = "https://uat.dwolla.com/oauth/rest";
    private final static String applicationKey = "";
