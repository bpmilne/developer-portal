Sending Money

Overview

The Send endpoint allows you to send money from an authenticated user (via OAuth) to a Dwolla account, or to the email address or phone number of someone who doesn't yet have a Dwolla account. Payments can be funded with a user's Balance, or with a Funding Source such as a US bank account, Dwolla line of Credit, or FiSync bank account.

Sending Money!

The Send endpoint requires you to specify an oauth_token, a string which represents a user's authorization, granted to your application to send money on the user's behalf. Our libraries let you execute this API call in as little as 3 lines of code!

If your application will only send money from a single account which you own, you can simply generate an OAuth token once using the token generator, and use it to send funds with. Note: Keep in mind that the access token generated using the token generator will expire in 1 hour, so to maintain access be sure to refresh authorization.

By default, if a Funding Source is not supplied (as the fundsSource parameter), funds are sent from the sender's account Balance which is a real-time funding source. Real-time funding sources clear in a matter of seconds.

require 'dwolla'

# Set a previously generated OAuth token
Dwolla::token = "Some OAuth token here"

transactionId = Dwolla::Transactions.send({
    :destinationId => '812-626-8794', 
    :amount => 1.00, 
    :pin => 9999 
    })

puts transactionId

# => "5118272"

from dwolla import transactions, constants

# Configure the library (change these)
constants.sandbox=False

constants.client_id = "..."
constants.client_secret = "..."
constants.access_token = "..."
constants.pin = 1234

# Example 1: Send $5.50 to a Dwolla ID.

print(transactions.send('812-197-4121', 5.50))

<?php
// Include the Dwolla REST Client
require '../lib/dwolla.php';

// Instantiate a new Dwolla REST Client
$Dwolla = new DwollaRestClient();

// Set a previously generated access token
$Dwolla->setToken("OAuth token here");

// Send $1.00 to a Dwolla ID with a $5.00 facilitator fee. 
$transactionId = $Dwolla->send($pin, '812-734-7288', 1.00);
?>

var dwolla = require('../lib/dwolla');

// Send $1.00 to a Dwolla ID, PIN: 1234
dwolla.send("OAuth token here", "1234", '812-626-8794', 1.00, function(err, tran) {
  if (err) console.log(err);
  console.log(tran);
});

Dwolla responds with a Transaction ID. This Id can later be supplied to the Transactions Lookup endpoint to retrieve more information about the payment, as well as its status. When the transaction has 'cleared', it will have a Status of processed. (Read more about Transaction Statuses)

Sending money from a bank funding source

In order to fund a payment with a ACH funding source (Funding source of ProcessingType == ACH), you will first need to list a user's funding sources using the Funding Sources / List endpoint. 

require 'dwolla'

# Set the payer's OAuth token
Dwolla::token = "Some OAuth token here"

# Get their funding sources!
sources = Dwolla::FundingSources.get

# Configure the library (change these)
constants.sandbox=False

constants.client_id = "..."
constants.client_secret = "..."
constants.access_token = "..."
constants.pin = 1234

# Example: Get a list of funding sources associated
# with the account under the current OAuth token

print(fundingsources.get())

<?php
// Include the Dwolla REST Client
require '../lib/dwolla.php';

// Instantiate a new Dwolla REST Client
$Dwolla = new DwollaRestClient();

// Set the payer's OAuth token
$Dwolla->setToken("OAuth token here");

// Fetch those funding sources!
$fundingSources = $Dwolla->fundingSources();
?>

var dwolla = require('dwolla');

// Fetch those funding sources!
dwolla.fundingSources("{Payer's OAuth token}", function(err, fundingSources) {
  if (err) { console.log(err); }
  console.log(fundingSources);
});

From there, provide the Id of the desired funding source as the fundsSource parameter in your call to Send().

Sending money from a Credit funding source

If the sender has Dwolla Credit, you can fund a payment from their line of credit by passing in a fundsSource of Credit in your Send() call. To tell if a user has a line of credit, use the above Funding Sources / List method to list their funding sources; Credit should appear as one of them.

It's important to note that not all Dwolla users may receive a credit-funded payment. Only merchants who are approved to do so may accept credit. Contact us to apply for approval!

Sending Money to a non-user

You can send money to someone who doesn't have an existing Dwolla account by sending to their email or phone number. They'll receive an email or SMS text message notifying them about the money received and inviting them to register a Dwolla account to claim it. Their money will be held for them until they claim it, but if they fail to claim the money within the sender's Reclaim period (an account-level setting which defaults to 30 days), the money will be returned to the sender and the status of the payment will be Reclaimed.
Note: The flow for claiming money sent to an email address will differ from money sent to a phone number. When money is sent to a phone number the recipient will need to signup for a full Dwolla account, whereas sending to an email address the recipient will signup for a Dwolla Direct account.

Clearing Times

If a payment is funded by real-time funding source, such as the sender's account balance, FiSync bank account, or line of credit, the transaction will clear in a matter of seconds.

If a payment is funded by a bank account, Dwolla uses the ACH network to fulfill it, which means the payment will require, on average, 3 - 5 business days to clear.

When a payment clears, the recipient has access to the funds sent.

Payments: A tale of two (or more) Transactions

Behind the scenes, each payment is actually represented as at least two Transactions: a debit from the sender's account, and a credit to the recipient's account. Each of these transactions has a unique Transaction.Id, and the recipient sees a different ID than the sender sees when looking at the same payment from their Dwolla.com dashboard.

Read the transactions section of our docs to learn more about how transactions work, and how to retrieve the recipient's transaction ID.
